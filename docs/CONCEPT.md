# ZombieIO Concept

## 1. High-Level Vision

ZombieIO is no longer a pure "zombie shooting" game.
The new direction is a hybrid survival game that combines:
- top-down action shooting
- roguelite progression
- light base building
- wave defense
- scavenging and extraction
- simple crafting and upgrade systems

The core fantasy is:
"Go out, loot resources, survive the horde, return to base, grow stronger, repeat."

## 2. Design Pillars

- Fast combat first: shooting and movement must always feel responsive.
- Multiple game layers: combat, loot, defense, and progression should support each other.
- Easy to expand: new weapons, enemies, maps, and systems should be data-driven when possible.
- Short-term fun, long-term growth: every run should feel good on its own, but also feed meta progression.
- Clear tension loop: explore, risk, extract, defend, upgrade.

## 3. Core Gameplay Loop

1. Select loadout or class.
2. Enter a map or mission zone.
3. Fight zombies, complete objectives, and collect resources.
4. Decide whether to keep pushing or extract early.
5. Return to base with loot.
6. Spend resources on upgrades, crafting, or base defense.
7. Unlock harder maps, stronger enemies, and new systems.

## 4. Genre Mix

The game should combine genres in a controlled way:

- Shooter: the main combat layer.
- Roguelite: random perks, run-based rewards, replayability.
- Survival: resource pressure and extraction decisions.
- Base Defense: protect the base during enemy waves or night events.
- Crafting: turn loot into gear, traps, or utility items.
- Light RPG: stats, perks, and character growth.

## 5. Proposed Game Structure

### 5.1 Mission Phase

The player leaves base and explores a zone.
Goals may include:
- collecting materials
- rescuing survivors
- destroying hives
- activating generators
- killing mini bosses

### 5.2 Extraction Phase

The player must decide when to leave.
The longer they stay, the better the rewards, but the danger grows.

### 5.3 Base Phase

Back at base, the player can:
- upgrade weapons
- unlock new perks
- build defensive structures
- craft consumables or traps
- prepare for the next mission

### 5.4 Defense Phase

At certain times, the base is attacked.
This creates a tower-defense-like pressure layer that forces the player to invest in defense, not only offense.

## 6. MVP Scope

The first playable version should be small and focused:

- one player character
- one main map
- one base hub
- 3 enemy types
- 2 weapons
- 1 boss
- 1 simple crafting system
- 1 upgrade screen
- 1 base defense wave
- 1 extraction objective

## 7. What This Game Is Not

- Not a complicated MMO
- Not a deep survival sim
- Not a full city builder
- Not a heavy story game

The project should stay small enough to build quickly, but rich enough to feel layered.

## 8. Content Expansion Strategy

Future content can grow by adding:
- new enemy variants
- new mission types
- new weapons and mods
- new perks and classes
- new base structures
- new maps and biomes
- special events and bosses

## 9. Final Goal

ZombieIO should become a compact hybrid action game with a strong loop:
combat -> loot -> upgrade -> defend -> repeat.
